# Support

## Usage and Configuration

Use the repository Issue templates for reproducible bugs and feature requests.
Include the application version, deployment method, database type, relevant
sanitized logs, and reproduction steps.

Do not post API keys, access tokens, cookies, database credentials, payment
records, or raw production configuration.

## Security Reports

Security vulnerabilities must not be posted in public Issues. Follow
`.github/SECURITY.md` and use private vulnerability reporting when available.
Existing QuantumNous security contacts and attribution remain preserved; the
fork maintainer should be included for fork-specific issues.

## Supported Versions

- `v0.1.6`: production compatibility baseline and critical-fix support line.
- `v0.2.x`: active development line after it is formally released.
- `v0.1.7` through `v0.1.14`: archived channel-routing line; direct upgrade to
  `v0.2.x` is not claimed without separate verification.
