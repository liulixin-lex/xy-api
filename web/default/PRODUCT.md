# Product

## Users

new-api is used by administrators, operators, and authenticated end users who manage lawful AI API gateway access, usage, billing, top-ups, provider routing, API keys, and account settings. Administrators work in dense settings, tables, and dashboards where scan speed, clear status, and predictable controls matter. End users need straightforward account, wallet, key, playground, and referral workflows without needing to understand internal infrastructure.

## Product Purpose

new-api is a unified AI API gateway and AI asset management system by QuantumNous. It aggregates many upstream model providers behind one authenticated interface, supports billing and quota accounting, and gives operators the controls needed for private or organization-level deployment. Success means users can confidently configure the system, understand balances and usage, and complete operational workflows with low ambiguity.

## Brand Personality

Restrained, capable, and trustworthy. The product should feel like a serious operations console: calm, efficient, legible, and direct, with enough polish to inspire confidence but no decorative complexity that slows a task.

## Anti-references

Avoid marketing-page composition inside authenticated surfaces, oversized hero treatments, decorative gradients, novelty controls, inconsistent table actions, and visual effects that compete with operational data. Avoid turning billing, settings, or referral workflows into promotional layouts; these surfaces should prioritize clarity, auditability, and predictable action states.

## Design Principles

1. Keep task surfaces quiet and information-dense so administrators can scan, compare, and act repeatedly.
2. Use the existing design system, table patterns, cards, drawers, badges, and form controls before introducing new UI vocabulary.
3. Make financial state explicit: distinguish pending, available, transferred, canceled, and historical amounts wherever those states affect user decisions.
4. Prefer clear progressive disclosure for complex settings, especially when a workflow mixes configuration, status, and historical data.
5. Preserve compliance and safety context around payments, top-ups, referral rewards, and quota movement.

## Accessibility & Inclusion

Follow WCAG 2.1 AA for text contrast, keyboard operation, focus visibility, and semantic form labels. Support reduced-motion preferences and avoid relying on color alone for reward status, validity, or enabled state. Maintain the existing i18n-first workflow so interface text remains available across supported locales.
