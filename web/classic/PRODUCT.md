# Product

## Register

product

## Users

Operators, administrators, and end users of an AI API gateway who need to manage API access, model routing, billing, logs, wallets, personal settings, and deployment status from a browser-based console. They are usually trying to complete operational tasks quickly and safely, often while comparing configuration, usage, quota, and provider status across multiple screens.

## Product Purpose

The classic frontend provides the established React console experience for New API. It supports AI gateway administration, user self-service, authentication, pricing visibility, usage analytics, and system configuration. Success means users can understand state, change settings, and complete account or API workflows without ambiguity or accidental destructive actions.

## Brand Personality

Practical, trustworthy, and efficient. The interface should feel like a mature operations console: dense enough for repeated work, clear enough for less technical users, and restrained enough that important status, actions, and warnings stand out.

## Anti-references

Avoid marketing-page composition inside the app shell, oversized decorative cards, decorative gradients, unnecessary glass effects, hidden critical controls, and visual experiments that reduce scan speed. Do not dilute or replace existing New API or QuantumNous identity, metadata, attribution, or protected project references.

## Design Principles

- Prioritize operational clarity over decoration.
- Keep repeated workflows compact, predictable, and easy to scan.
- Surface risk, state, and next actions in plain language.
- Preserve established classic frontend conventions unless a change clearly improves usability.
- Treat configuration and billing screens as high-stakes surfaces that require precise labels, stable layout, and reversible actions.

## Accessibility & Inclusion

Target clear keyboard navigation, readable text contrast, reduced ambiguity in action labels, and layouts that remain usable on common desktop and mobile breakpoints. Motion should not be required to understand or operate the interface, and color should not be the only signal for status or errors.
